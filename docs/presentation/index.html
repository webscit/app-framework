↩ Last-message replay
Both the Python bus and the TS client cache the latest event per channel. New subscribers render the last value instantly — no polling, no blank state on mount.
⌖ Channel-first resolution
WidgetRegistry matches channelPattern glob → MIME type refinement → priority sort. Same model as VS Code editor resolution.
⏺ Persisted layout
Zustand + localStorage with schema version migration. The layout is a serialisable document, not ephemeral React state.
The star feature

AI layout assistant.

sequenceDiagram actor U as User participant C as AIChatPanel participant B as POST /ai/layout participant AI as OpenRouter LLM participant S as ApplicationShell U->>C: "Show sine chart with frequency sliders" C->>B: POST {prompt, registry, current_layout} B->>AI: system prompt + widget catalog AI-->>B: tool_call: get_widget_details("Chart") B-->>AI: {channelPattern, parameters schema} AI->>B: {layout: ShellLayout, explanation} B->>B: validate widget types & IDs B->>C: 200 {layout, explanation} C->>U: LayoutDiffViewer — current vs proposed U->>C: Approve ✓ C->>S: setLayout(applyNonTogglableCorrection)
Approved-only history
Rejected proposals are never included in subsequent requests. The AI's context stays clean — only turns the user actually accepted flow back to the model.
Auto-retry with correction hint
If the AI hallucinates an unregistered widget type, the backend retries once with a correction message listing valid widget names. Falls back to 422 on second failure.
